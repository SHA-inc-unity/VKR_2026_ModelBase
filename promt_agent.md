# promt_agent

Краткий рабочий дневник агента по репозиторию ModelLine.

## Правила ведения

- Читать этот файл перед началом любой работы после базовых workflow-документов.
- После завершения работы добавлять короткую запись: что выяснили, что сделали, что осталось.
- Писать кратко и по делу, без длинной исторической ленты.

- Launcher теперь в `restart` автоматически восстанавливает отсутствующий `.env` из `.env.example`, а dangling image prune вынесен из child fan-out в один parent cleanup-pass, чтобы parallel deploy не падал на `a prune operation is already running`.
- `microservice_analitic/docker-compose.yml`: встроенный Redis больше не публикуется на host `6379`; он остаётся только внутренней зависимостью `api`/`scheduler`, чтобы backend-host не ловил port conflict при auto-deploy.
- Launcher cleanup усилен межпроцессным lock-ом вокруг `docker image prune`, чтобы даже при нескольких параллельных child-процессах cleanup не конфликтовал между собой.
- Старый split-transport код, compose-сервисы и отдельные markdown-гайды удалены из активной схемы.
- Актуальные операционные документы: root `README.md`, `STRUCTURE.md`, `microservicestarter/README.md`, `microservice_admin/README.md`, `microservice_infra/README.md`.
- `microservice_admin/src/app/api/health/route.ts`: split-mode health response снова совпадает с `InfraHealthResponse`; `npm run build` для admin проходит.
- `microservice_infra/docker-compose.yml`: backend HTTPS facade на `:8443` больше не падает без ручных `tls.crt` / `tls.key`; one-shot `nginx-cert-init` автогенерирует self-signed cert в `ADMIN_BACKEND_CERTS_DIR`, если каталог пустой.
- `microservice_admin/docker-compose.yml` + `src/lib/backendClient.ts`: split admin-head получил `ADMIN_BACKEND_TLS_INSECURE`; `admin-online` по умолчанию принимает autogenerated self-signed backend cert, а trusted-cert сценарий остаётся через `ADMIN_BACKEND_TLS_INSECURE=0`.
- Актуальные env/docs очищены от legacy transport-формулировок: в живых конфигурациях split deployment больше описывается только через backend facade `:8443` и обычные private/public IP или DNS.
- `microservicestarter/start.sh` и `restart.sh`: bash launcher теперь заранее проверяет publish-ports (`8443`, `8501`, `7510`, `7520`, infra ports, `ADMIN_HTTP_PORT`, `ADMIN_HTTPS_PORT`) и падает с явным сообщением, если их держит чужой контейнер/процесс. `stop.sh` по-прежнему управляет только compose-проектами ModelLine и не удаляет сторонние контейнеры.
- `microservice_infra/nginx/nginx.conf`: split facade `/api/admin/*` был сломан alias-ом `gateway:5020`; реальный upstream в сети `modelline_net` — `gateway-service:5020`. Из-за этого `8443` мог отдавать `502`, даже когда прямой `http://<host>:7520/health` уже был healthy.
- `microservicestarter` preflight check на порты в bash-скриптах не должен ломать CI/CD restart: сравнение container IDs теперь учитывает long/full IDs и считает уже работающие контейнеры того же compose-проекта своими, а не внешними.
- `microservicestarter` preflight parser теперь понимает Docker publish ranges (`9000-9001->9000-9001`), иначе свой `minio` не находился через `docker ps`, а `ss` видел только `docker-proxy` и ложноположительно блокировал повторный `restart`.
- Split admin diagnostics усилены: admin `backendClient`/`/api/kafka` пробрасывает `status`, `code`, `detail`, `correlationId` и отдельно классифицирует `401/403`, `502`, `504` и network failures.
