- `microservice_data/src/DataService.API/{Markets/KrakenApiClient.cs,Markets/KrakenRateLimiter.cs,Program.cs}`, `microservice_data/{README.md,STRUCTURE.md,EXCHANGE_APIS.md}`: закрыт live regression после per-exchange 4-slot ingest pool для Kraken. Root cause был не в scheduler-е, а в public REST path: несколько Kraken jobs одновременно запускали paged OHLC fetch и быстро ловили `EGeneral:Too many requests`. В Kraken client добавлен process-local limiter (общий inflight gate + inter-request spacing) и retry/backoff на Kraken-specific throttle ошибки (`Too many requests`, `Rate limit exceeded`, `Throttled: <unix>`). Live validation после rebuild: четыре `cmd.data.dataset.jobs.start` для `kraken_btcusdt_{120m,240m,720m,1d}` были приняты; `120m` и `240m` завершились `succeeded`, а `720m` и `1d` остались healthy `running` вместо мгновенного terminal fail.

- `microservice_data/src/DataService.API/{Jobs/JobHandlers.cs,Markets/IMarketDataClient.cs,Markets/BinanceApiClient.cs,Bybit/BybitApiClient.cs,Markets/KrakenApiClient.cs}`, `microservice_data/{README.md,STRUCTURE.md}`: проверен live incident со «застывшими» dataset jobs на `fetch_oi`. Root cause оказался не в deadlock/job-runner, а в progress-модели: jobs в Postgres продолжали heartbeat (`updated_at` age ~2–3 s), но `IngestJobHandler` держал фиксированный `47%`, потому что market client interface умел page-progress только для `fetch_klines`. В `FetchOpenInterestAsync` добавлен optional `onPageDone`, Bybit/Binance теперь репортят completed OI pages, а handler публикует промежуточный `fetch_oi` progress вместо вечного `waiting for open interest`.

- `microservice_data/src/DataService.API/{Jobs/JobHandlers.cs,Markets/KrakenApiClient.cs}`, `microservice_admin/src/{hooks/useDatasetJobs.ts,app/download/page.tsx,app/api/queue/history/route.ts,components/DatasetJobsPanel.tsx}`, `microservice_data/EXCHANGE_APIS.md`, `microservice_data/{README.md,STRUCTURE.md}`, `microservice_admin/{README.md,STRUCTURE.md}`, `docs/agents/services/microservice_data.md`: закрыт следующий срез после ingest-starvation. В admin live dataset jobs `failed/canceled` автоубираются через 10 секунд, terminal completions best-effort пишутся в SQLite-backed queue history через `POST /api/queue/history`, а Download-page одновременно сохраняет ingest errors в локальный Action History, чтобы transient errors не раздували layout. Добавлен `microservice_data/EXCHANGE_APIS.md` — короткая памятка по Kraken / Bybit / Binance API и их реальным ограничениям для ModelLine.

- `microservice_data/src/DataService.API/{Jobs/DatasetJobRunner.cs,Jobs/JobHandlers.cs,Kafka/KafkaProducer.cs,Markets/KrakenApiClient.cs,Markets/BinanceApiClient.cs,Bybit/BybitApiClient.cs}`, `microservice_data/{README.md,STRUCTURE.md}`: закрыт dataset ingest starvation/stall вокруг Kraken. Root cause был двойной: один общий ingest cap `4` позволял одной бирже занять все runner slots, а progress publish ждал Kafka delivery и мог подвесить job handler на `fetch_klines`. Scheduler переведён на `4` ingest-slot'а на каждую биржу + отдельный heavy gate `1` на `1m`/`3m` для каждой биржи, progress events стали best-effort, Kraken ingest теперь клипует request window к reachable history (последние 720 base candles) и fast-fail'ит полностью недостижимые окна, а Bybit/Binance/Kraken clients читают response body целиком под общим HTTP timeout. Runtime validation: `dotnet build src/DataService.API/DataService.API.csproj` проходит, rebuilt `microservice_data` healthy, requeued `bybit` `720m` job завершается сразу даже при одновременно running `kraken` `720m`, то есть биржи больше не блокируют друг друга общим ingest-пулом.
