# Agent Docs

Этот каталог содержит Markdown-структуру, на которую должны опираться агенты разработки.

## Назначение

- дать агенту единый маршрут чтения перед работой с кодом
- зафиксировать, какие документы являются источником истины
- определить, какие Markdown-файлы нужно обновлять после изменений
- сократить число ситуаций, когда код и документация расходятся

## Порядок чтения

1. [../../AGENTS.md](../../AGENTS.md)
2. [WORKFLOW.md](WORKFLOW.md)
3. [DOCS_MAP.md](DOCS_MAP.md)
4. [../../promt_agent.md](../../promt_agent.md)
5. [services/README.md](services/README.md)
6. README.md и, если существует, STRUCTURE.md затронутого сервиса
7. Профиль затронутого сервиса в `services/`

## Состав каталога

- [WORKFLOW.md](WORKFLOW.md) — обязательный порядок действий до и после работы
- [DOCS_MAP.md](DOCS_MAP.md) — карта Markdown-источников по репозиторию
- [../../promt_agent.md](../../promt_agent.md) — краткий рабочий дневник агента
- [CHANGE_LOG.md](CHANGE_LOG.md) — журнал заметных изменений для агентной работы
- [services/README.md](services/README.md) — индекс сервисных профилей
- `services/*.md` — краткие сервисные опоры для агента

## Базовое правило

Если агент меняет код, он обязан в рамках той же работы привести Markdown-документы в актуальное состояние.
