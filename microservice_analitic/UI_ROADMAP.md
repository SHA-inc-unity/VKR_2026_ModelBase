# UI Roadmap — ModelLine

Задачи по улучшению интерфейса (без расширения функционала).
Приоритет: высокий / средний / низкий.

---

## Высокий приоритет

### 1. Единое состояние symbol/timeframe между страницами
**Проблема:** пользователь выбрал ETHUSDT на странице модели, открывает Загрузку данных — там снова BTCUSDT.
**Решение:** обе страницы читают `ds_symbol`/`ds_timeframe` из одного ключа store; при переходе с model_page на download_page передавать символ через query params (`st.query_params`).

### 2. Индикатор "таблица существует" на странице модели
**Проблема:** пользователь не знает, загружены ли данные, пока не нажмёт кнопку.
**Решение:** при загрузке страницы делать быстрый `SELECT COUNT(*)` с таймаутом 2 с; показывать зелёный/красный индикатор рядом с полем Symbol.

### 3. Кеш target-кандидатов не инвалидируется после автозагрузки
**Проблема:** после авто-скачивания данных с Bybit selectbox "Целевая переменная" остаётся пустым до следующего F5.
**Решение:** в `_load_and_cache_data_auto` уже есть `st.session_state.pop("_target_candidates_cache")` — проверить что `st.rerun()` вызывается корректно.

### 4. Дублирование блока "Статус базы данных" на обеих страницах
**Проблема:** одинаковый блок с Host/Port/Database/Status рендерится на model_page и download_page отдельно.
**Решение:** вынести в `services/ui_components.py` как `render_db_status(db_config, db_status)`.

## Средний приоритет

### 5. Страница загрузки данных — результат загрузки теряется при F5
**Проблема:** `ds_last_download` хранится только в session_state — после перезагрузки пропадает.
**Решение:** сохранять в store под ключом `ds:last_download:{table_name}` с timestamp; показывать "последняя загрузка: 2 часа назад".

### 6. Grid Search — таблица результатов не сортируется в UI
**Проблема:** пользователь не может кликнуть на заголовок и отсортировать по RMSE или direction accuracy.
**Решение:** `st.dataframe` уже поддерживает сортировку — убедиться что `column_config` не отключает её; убрать лишние `disabled=True`.

### 7. Spinner "Загрузка данных..." перекрывает всю страницу
**Проблема:** при долгой загрузке пользователь не видит прогресс.
**Решение:** заменить `st.spinner` на `st.status` с `expanded=True` там где есть прогресс-бар (автозагрузка, download_page).

### 8. Кнопка "← Назад" дублируется логикой на каждой странице
**Проблема:** копипаст `if st.button("← Назад"): st.switch_page("app.py")` на 2 страницах.
**Решение:** вынести в `services/ui_components.py` как `render_back_button()`.

### 9. compare_page — нет состояния "нет сессий"
**Проблема:** если моделей нет, страница показывает пустой список без объяснений.
**Решение:** добавить `st.info("Обучите хотя бы одну модель на странице модели")` + ссылку `st.page_link`.

### 10. Реестр моделей — нет кнопки "Загрузить параметры в форму"
**Проблема:** пользователь видит best_params версии в реестре, но не может одним кликом применить их к текущей форме обучения.
**Решение:** кнопка "Применить параметры" → записывает в session_state и сохраняет через `save_grid_params_config`.

## Низкий приоритет

### 11. Страница модели слишком длинная — нет якорей
**Проблема:** страница с 4 вкладками + параметры + статус БД требует много скролла.
**Решение:** вынести блок "Статус БД" в collapsible expander (по умолчанию свёрнут если connected).

### 12. download_page — "Сырые данные" всегда отображает все столбцы
**Проблема:** таблица с 60+ столбцами неудобна.
**Решение:** по умолчанию показывать только `[timestamp_utc, open, high, low, close, volume, rsi, index_price]`; остальные — через multiselect.

### 13. model_page — MLflow expander открыт каждый раз
**Проблема:** если MLflow отключён, expander всё равно занимает место.
**Решение:** показывать expander только если `use_mlflow=True`, иначе — простой checkbox без дополнительных полей.

### 14. Нет тостов/уведомлений об успешном сохранении настроек
**Проблема:** после нажатия "Сохранить настройки" появляется `st.success` который исчезает только при следующем rerun.
**Решение:** использовать `st.toast("Настройки сохранены", icon="✅")` — не блокирует страницу.

### 15. app.py (главная) — нет информации о состоянии системы
**Проблема:** главная страница показывает только навигацию.
**Решение:** добавить мини-статус: подключена ли БД, сколько таблиц с данными, сколько обученных моделей в реестре.
